# Deploy to Render

This project is ready for Render using `render.yaml`.

## 1) Push code to GitHub

The repository is already linked:

- `https://github.com/karthiksai22102005-ship-it/College_admin.git`

Push latest changes:

```bash
git add .
git commit -m "Prepare Render deployment startup bootstrap"
git push
```

## 2) Create service from Blueprint

1. Open Render dashboard.
2. Click **New** -> **Blueprint**.
3. Select your GitHub repo: `College_admin`.
4. Render will detect `render.yaml` and create:
   - web service: `college-admin`
   - postgres database: `college-admin-db`

## 3) Required environment variables

`render.yaml` already sets:

- `PYTHON_VERSION=3.11.9`
- `DB_BACKEND=postgres`
- `SESSION_COOKIE_SECURE=true`
- `SESSION_COOKIE_SAMESITE=Lax`
- `ALLOW_LEGACY_ADMIN_LOGIN=false`
- `SECRET_KEY` generated by Render
- `DATABASE_URL` linked from Render database

No manual env var setup is required unless you need custom values.

## 4) First deploy checks

After deploy, verify:

1. Health endpoint returns OK:
   - `https://<your-render-service>.onrender.com/health`
2. App login page loads:
   - `https://<your-render-service>.onrender.com/`
3. Render logs show Gunicorn started without import/runtime errors.

## 5) Common issue

If deploy succeeds but login fails, seed your initial admin/faculty data from a Render shell:

```bash
python init_data.py
python seed_users.py
python seed_faculty.py
```

Run only the scripts that match your intended data setup.
